# 컴파일러 최적화 기법 상세 명세

## 1️⃣ Constant Folding (상수 폴딩)

### 개념
컴파일 타임에 상수로만 이루어진 표현식을 계산하여 단일 상수값으로 치환합니다.

### 알고리즘
```
1. AST 순회
2. 각 노드의 자식이 모두 상수인지 확인
3. 맞으면 식 계산 후 상수 노드로 치환
4. 재귀적으로 부모 노드에 적용
```

### 지원 연산
- 산술: `+`, `-`, `*`, `/`, `%`
- 논리: `&&`, `||`
- 비트: `&`, `|`, `^`

### 예제
```
입력:  5 + 3 * 2
AST:   BinOp(+, 5, BinOp(*, 3, 2))

처리:
1. BinOp(*, 3, 2) → 6 (상수 계산)
2. BinOp(+, 5, 6) → 11 (상수 계산)

출력:  11
```

### 제약
- 부동소수점 정밀도 손실 가능
- 오버플로우 처리 필요

### 성능
- 시간복잡도: O(n)
- 공간복잡도: O(1)

---

## 2️⃣ CSE (Common Subexpression Elimination)

### 개념
동일한 부분식이 여러 번 나타날 때, 첫 계산 결과를 재사용합니다.

### 알고리즘
```
1. 모든 표현식을 정규화 (정렬된 형태)
2. 정규화된 표현식을 맵에 저장
3. 같은 표현식이 2회 이상 나타나면 마크
4. 임시 변수 생성 및 재사용
```

### 정규화 규칙
- 교환법칙 적용: `a + b` = `b + a`
- 비교 연산: `a > b` ≠ `b > a` (비교환)
- 함수: 순수 함수만 CSE 대상

### 안전성
- 부작용 있는 함수 제외 (제약)
- 메모리 접근 변수 제외
- 상수/변수만 CSE 적용

### 예제
```
입력:
a = x + y
b = x + y
c = x + y

정규화: (x+y) = 3회 출현

출력:
__cse_0 = x + y
a = __cse_0
b = __cse_0
c = __cse_0
```

### 성능
- 시간복잡도: O(n log n) (정렬)
- 공간복잡도: O(n)

---

## 3️⃣ Dead Code Elimination (죽은 코드 제거)

### 개념
실행되지 않거나 사용되지 않는 코드를 AST에서 제거합니다.

### 죽은 코드 분류

#### 1. 도달 불가능 코드
```
if (false) {
  // 절대 실행되지 않음
}

return 42;
unreachable_code(); // 도달 불가
```

#### 2. 미사용 변수 할당
```
x = 5;  // x가 사용되지 않으면 제거
y = expensive_func();  // 부작용 있으면 보존
```

#### 3. 불가능한 조건
```
if (0) { ... }        // 제거
while (false) { ... } // 제거
```

### 알고리즘
```
1. 변수 사용 분석 (정의/사용)
2. 조건 상수성 판단
3. 제어 흐름 분석 (도달성)
4. 해당 코드 제거 또는 단순화
```

### 안전성
- 부작용 있는 코드는 보존 (함수 호출)
- 예외 처리 고려
- 메모리 접근 보존

### 예제
```
입력:
if (true) {
  x = 10;
} else {
  y = 20;  // 도달 불가
}
z = unused;  // 미사용

출력:
x = 10;
```

### 성능
- 시간복잡도: O(n)
- 공간복잡도: O(n)

---

## 4️⃣ Function Inlining (함수 인라이닝)

### 개념
작은 함수의 호출을 함수 본체로 직접 치환하여 호출 오버헤드를 제거합니다.

### 인라이닝 결정 기준
```
인라이닝 조건:
- 함수 복잡도 < 50 (AST 노드 수)
- 호출 횟수 ≤ 3회
- 재귀 함수 아님
- 부작용 없음
```

### 알고리즘
```
1. 함수 정의 수집 (맵 구성)
2. 각 함수 호출 횟수 계산
3. 인라이닝 조건 확인
4. 매개변수 치환 및 본체 복사
```

### 예제
```
입력:
fn double(x) { return x * 2; }
a = double(5);
b = double(10);

처리:
1. double은 간단 + 2회 호출 ✓
2. 본체 복사: (5 * 2), (10 * 2)

출력:
a = 5 * 2;     // 10
b = 10 * 2;    // 20
```

### 매개변수 치환
```
fn add(a, b) { return a + b; }
result = add(3, 4);

인라이닝:
result = 3 + 4;  // 매개변수 a→3, b→4 치환
```

### 성능
- 시간복잡도: O(n × m) (n: 함수 수, m: 본체 크기)
- 공간복잡도: O(n × m) (복사 본체)

## 5️⃣ LICM (Loop-Invariant Code Motion)

### 개념
루프 내에서 반복 계산되지만 결과가 변하지 않는 코드를 루프 밖으로 이동합니다.

### 루프 불변식 판정
```
식이 루프 불변식 ⟺ 식이 루프 변수를 포함하지 않음

루프 불변식:
- 상수
- 루프 외부 변수
- 루프 불변식의 조합

루프 변수식:
- 루프 변수 자체
- 루프 변수를 포함하는 식
```

### 알고리즘
```
1. 루프 변수 식별
2. 루프 본체의 모든 식 분석
3. 루프 불변식 찾기
4. 루프 전에 이동
```

### 예제
```
입력:
for (i in 0..n) {
  x = a + b;     // 루프 불변
  y = i * 2;     // 루프 변수식
}

처리:
LICM: x = a + b를 루프 전으로 이동

출력:
x = a + b;  // 1회만 계산
for (i in 0..n) {
  y = i * 2;
}
```

### 안전성
- 종속성 분석 필요
- 부작용 있는 코드는 이동 금지

### 성능
- 시간복잡도: O(n)
- 공간복잡도: O(1)
- 효과: 루프 반복만큼 감소 (n배)

---

## 6️⃣ Loop Unrolling (루프 펼침)

### 개념
루프 반복을 펼쳐서 분기 횟수와 루프 오버헤드를 감소시킵니다.

### 펼침 계수 결정
```
펼침 계수 = min(
  루프 카운트,
  MAX_UNROLL_FACTOR (8),
  MAX_BODY_SIZE / 복잡도
)

예: 8회 반복, 본체 간단 → 펼침 계수 8
```

### 알고리즘
```
1. 루프 카운트 추출 (정적 알려진 값)
2. 펼침 인수 결정
3. 루프 변수를 반복별로 치환
4. 펼쳐진 명령 시퀀스 생성
```

### 예제
```
입력:
for (i in 0..3) {
  sum += arr[i];
}

펼침 (인수 4):
sum += arr[0];
sum += arr[1];
sum += arr[2];
sum += arr[3];

효과:
- 분기 3번 제거
- 루프 오버헤드 제거
```

### 제약
- 반복 횟수가 미리 알려져야 함
- 루프 본체가 너무 복잡하면 불가
- 코드 크기 증가 (trade-off)

### 성능
- 시간복잡도: O(n × k) (k: 펼침 인수)
- 공간복잡도: O(n × k) (코드 크기)
- 효과: 분기 오버헤드 k배 감소

## 7️⃣ Strength Reduction (강도 감소)

### 개념
비싼 연산을 더 싼 연산으로 치환합니다.

### 강도 감소 규칙

#### 곱셈/나눗셈
```
x * 0   → 0
x * 1   → x
x * 2^n → x << n    (시프트: 1-2 사이클)
x / 1   → x
x / 2^n → x >> n
```

#### 덧셈/뺄셈
```
x + 0 → x
x - 0 → x
0 - x → -x
```

#### 루프 유도 변수
```
for i in 0..n:
  x = i * 4  →  x = x + 4  (곱셈 → 덧셈)
```

### 비용 모델
```
연산 비용 (사이클):
- 덧셈: 1
- 시프트: 1
- 곱셈: 3-4
- 나눗셈: 10-40

감소 효과: 곱셈 → 시프트(3배 빠름)
```

### 예제
```
입력:
a = i * 8;
b = j / 16;

변환:
a = i << 3;   // 3배 빠름
b = j >> 4;   // 20배 빠름
```

### 성능
- 시간복잡도: O(n)
- 공간복잡도: O(1)
- 효과: 연산별로 다름 (1-40배)

---

## 8️⃣ Tail Call Optimization (꼬리 호출 최적화)

### 개념
함수의 마지막이 재귀 호출인 경우, 스택 프레임을 재사용하여 스택 오버플로우를 방지합니다.

### 꼬리 위치 판정
```
표현식이 꼬리 위치 ⟺
결과가 즉시 반환되는 위치

꼬리 위치:
- return 문의 표현식
- 블록의 마지막 문
- if의 양쪽 분기

비-꼬리 위치:
- 변수 할당의 우변
- 함수 인자
- 산술 연산 피연산자
```

### 알고리즘
```
1. 재귀 함수 식별
2. 꼬리 위치의 호출 찾기
3. TCO 마킹
4. 런타임: 루프로 변환
```

### 예제
```
입력:
fn fib(n, a=0, b=1):
  if n <= 1:
    return b
  return fib(n-1, b, a+b)  // 꼬리 위치

처리:
1. fib → fib 재귀 호출
2. return 문 → 꼬리 위치 ✓
3. TCO 적용

효과:
- 스택 사용: O(n) → O(1)
- 스택 오버플로우 방지
```

### 스택 프레임 재사용
```
일반 재귀:
Stack: [..., fib(n), fib(n-1), fib(n-2), ...]

TCO 적용:
Stack: [..., fib(n)]  // 루프 변수 업데이트만
```

### 성능
- 시간복잡도: O(1) (프레임 재사용)
- 공간복잡도: O(1) (스택)
- 효과: 무제한 깊이 재귀 가능

## 9️⃣ Vectorization (벡터화)

### 개념
스칼라 루프를 벡터 연산(SIMD)으로 변환하여 병렬성을 활용합니다.

### 벡터화 가능 패턴
```
for i in 0..n:
  c[i] = a[i] + b[i]

벡터 형식:
c[0:n] = a[0:n] + b[0:n]  // 4-8개 원소 동시 처리
```

### 벡터화 조건
```
1. 반복 횟수 알려짐 (≥ 4)
2. 루프 본체 단순 (배열 접근만)
3. 종속성 없음
4. 분기 없음
```

### 알고리즘
```
1. 루프 분석
2. 벡터화 가능성 판정
3. 배열 접근 패턴 확인
4. 벡터 지시 생성
```

### 예제
```
입력:
for (i in 0..8):
  result[i] = input[i] + scale;

벡터화 (4개 원소):
result[0:3] = input[0:3] + [scale, scale, scale, scale];
result[4:7] = input[4:7] + [scale, scale, scale, scale];

효과:
- 반복 8회 → 2회 (4배 빠름)
- SIMD 유닛 활용
```

### SIMD 성능
```
연산 비용 (스칼라 vs 벡터):
- 덧셈: 1 cycle → 1 cycle (4개 동시)
- 곱셈: 3 cycles → 3 cycles (4개 동시)
- 로드: 3 cycles → 3 cycles (128비트)

속도 향상: 4배 (128비트 SIMD)
           8배 (256비트 AVX)
          16배 (512비트 AVX-512)
```

### 제약
- 컴파일러/CPU 벡터 지원 필요
- 메모리 정렬 요구
- 분기 제거 필수

### 성능
- 시간복잡도: O(n/k) (k: 벡터 폭)
- 공간복잡도: O(1)
- 효과: 4-16배 속도 향상

## 📊 최적화 효과 비교

| 기법 | 크기 감소 | 속도 향상 | 적용 난이도 | 적용 범위 |
|------|---------|---------|-----------|---------|
| Constant Folding | 20-30% | 5-10% | 쉬움 | 모든 식 |
| CSE | 10-15% | 3-8% | 중간 | 중복 식 |
| Dead Code | 5-20% | 2-5% | 쉬움 | 사용 안 함 |
| Inlining | 5-30% | 5-50% | 중간 | 작은 함수 |
| LICM | 5-15% | 10-100% | 중간 | 루프 불변 |
| Loop Unroll | 0-10% | 10-40% | 중간 | 정적 루프 |
| Strength Reduce | 1-5% | 20-40% | 쉬움 | 연산 |
| Tail Call | 0% | 100% (deep) | 쉬움 | 재귀 |
| Vectorization | 0-20% | 4-16x | 어려움 | 배열 루프 |

## 🔄 최적화 상호작용

최적화 기법들은 서로 상호작용하여 추가 최적화 기회를 만듭니다:

```
Constant Folding
    ↓ (상수 생성)
CSE (더 많은 중복 발견)
    ↓ (중복 제거)
Dead Code (미사용 코드 생성)
    ↓ (정리)
LICM (더 간단한 루프)
    ↓ (반복 최적화)
...
```

이것이 반복 최적화가 중요한 이유입니다!
